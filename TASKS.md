# TASKS — текущий этап разработки

## Статус релизного цикла

Этап: **Hardening / Content Integration**

Цель этапа: стабилизировать ядро, завершить контентную настройку маршрутов/активностей, подготовить систему к масштабированию на большее число областей.

---

## Выполнено

- [x] Событийная архитектура area-driven без heartbeat NPC.
- [x] Тик области с защитой токеном (`al_tick_token`).
- [x] Плотный реестр NPC + swap-remove.
- [x] Синхронизация реестра при активации и периодически в тике.
- [x] RESYNC при активации области и при позднем спавне NPC.
- [x] Route repeat (`AL_EVT_ROUTE_REPEAT`) и управление циклом маршрута.
- [x] Базовая поддержка training/bar pair зависимостей.
- [x] Деактивация области и скрытие NPC при `player_count = 0`.

## В работе

- [ ] Валидация контента waypoint-маршрутов по всем игровым областям.
- [ ] Диагностика переполнения реестра (`AL_MAX_NPCS`) на live-контенте.
- [ ] Уточнение таблицы активностей для route-required сценариев.

## Запланировано (ближайшие)

1. Добавить диагностические сообщения/метрики при отказе регистрации NPC из-за лимита.
2. Подготовить рекомендации по безопасному увеличению `AL_MAX_NPCS`.
3. Формализовать чеклист для контент-дизайна маршрутов (теги, индексы, дубликаты).
4. Подготовить регрессионный набор сценариев для тестовой области (enter/exit/slot switch/death/respawn).

## Запланировано (после стабилизации)

- Профилирование первого кеширования маршрутов на крупных областях.
- Опциональный lazy-refresh для маршрутов без потери корректности поведения.
- Расширение отчётности по отладке (пер-область, пер-NPC).
