# AUDIT — текущее QA-резюме

Дата актуализации: 2026-03-11

## Общий вывод

Система соответствует целевой модели “один тик на область, ноль фоновой активности NPC”.
Критичных архитектурных расхождений с текущей кодовой базой не обнаружено.

## Проверенные зоны

- Area lifecycle: activate/tick/deactivate.
- Реестр NPC: регистрация, удаление, синхронизация.
- Event protocol: `AL_EVT_SLOT_*`, `AL_EVT_RESYNC`, `AL_EVT_ROUTE_REPEAT`.
- Route/activity связка и fallback-поведение.
- Pair-механики (training/bar) при spawn/death.
- Защита от двойного decrement через `al_exit_counted`.

## Актуальные риски

1. **Лимит `AL_MAX_NPCS=100`**
   - Риск: тихое выпадение части NPC из управления.
   - Приоритет: высокий для крупных/динамических областей.

2. **Первичное кеширование маршрутов**
   - Риск: пиковая нагрузка при первом входе игрока в большую область.
   - Приоритет: средний.

3. **Качество контентной разметки waypoint**
   - Риск: некорректные теги/индексы ломают route-driven поведение.
   - Приоритет: средний.
