# PycukSystems / Ambient Life v2 (NWN2)

Актуальная документация по текущему состоянию системы Ambient Life.

## Текущее состояние проекта

Стадия: **стабилизированное ядро** (production-ready foundation).

Что уже реализовано и используется:
- событийная архитектура без heartbeat у NPC;
- один тик на область (`AL_TICK_PERIOD = 45.0`) только при наличии игроков;
- плотный реестр NPC на области (`al_npc_0..al_npc_99`) со swap-remove;
- синхронизация и пересборка реестра на уровне области;
- маршруты по waypoint-тегам с кешем на области;
- обработка повторов маршрута через `AL_EVT_ROUTE_REPEAT`;
- поддержка парных сценариев: training-пара и bar-пара;
- безопасная деактивация при уходе последнего игрока.

## Ключевые ограничения

- Жёсткий лимит управляемых NPC на область: **100** (`AL_MAX_NPCS`).
- Система требует корректной привязки event-скриптов в toolset.
- Для маршрутных активностей нужны валидные waypoint-теги и/или индексированные пути.

## Состав репозитория

- `scripts/` — рабочие NWScript-скрипты Ambient Life.
- `tools/nwn2_compiler/` — локальный компилятор и stock scripts.
- `archive/` — архив старых/предстабильных материалов (исторические, не канонические).
- `ARCHITECTURE.md` — детальная архитектура и протокол событий.
- `INSTALLATION.md` — установка и интеграция в модуль.
- `TASKS.md` — актуальный backlog по этапу разработки.
- `AUDIT.md` — текущее QA-резюме и зоны риска.

## Быстрый старт

1. Подключить include/скрипты из `scripts/` в модуль NWN2.
2. Назначить обработчики событий (Area/NPC/Module) согласно `INSTALLATION.md`.
3. Настроить маршрутные waypoint-теги и парные locals на нужных областях.
4. Скомпилировать и проверить сценарии входа/выхода игроков и смены слотов времени.
