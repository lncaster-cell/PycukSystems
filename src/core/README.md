# `src/core` — контракт каталога

## Что будет размещаться
- Event-driven ядро runtime: маршрутизация входящих событий движка и диспетчеризация в модули.
- Общие runtime-сервисы: lifecycle модулей (`init/start/stop/reload`), базовые guardrails, общие контракты метрик/health.
- Переиспользуемые low-level примитивы, не привязанные к конкретной механике (например, общие state/queue helpers).

## Критерий готовности
Каталог считается готовым к первой фазе, когда:
1. Реализован минимальный каркас ядра событий и lifecycle-контрактов для модулей.
2. Контракты и точки расширения используются минимум одним прикладным модулем (`npc_behavior`) без дублирования core-логики.
3. Пройдены базовые perf-check'и на отсутствие деградации относительно baseline для event routing.
