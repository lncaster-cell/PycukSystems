# Дизайн-документ (Диздок)

## 1. Цель
Подготовить техническую основу для разработки системы: воспроизводимое окружение, единые правила запуска и стартовую архитектурную рамку.

## 2. Контекст
Репозиторий инициализирован и готов к наполнению. На текущем этапе приоритет — стандартизировать процесс старта разработки для команды.

## 3. Задачи подготовки

1. Создать локальное изолированное окружение разработки.
2. Зафиксировать минимальные артефакты документации.
3. Подготовить основу для масштабирования (скрипты, структура, единые соглашения).

## 4. Принятые решения

### 4.1 Окружение разработки
- Используется `python3` + `venv` для изоляции зависимостей.
- Генерируется локальное окружение `.venv/`.
- Добавлен скрипт `scripts/setup_env.sh` для автоматизации.

### 4.2 Документация
- Основные шаги запуска зафиксированы в `README.md`.
- Технический дизайн и подготовка зафиксированы в этом документе.

### 4.3 Структура проекта
- `src/` — прикладной код.
- `tests/` — тесты.
- `docs/` — документация.
- `scripts/` — автоматизация.

## 5. Функциональные требования этапа
- Разработчик должен за <= 5 минут подготовить рабочее окружение.
- Инструкции запуска должны быть понятны без устных комментариев.

## 6. Нефункциональные требования
- Повторяемость шагов на Linux/macOS.
- Минимальный входной порог для новых участников.
- Возможность интеграции в CI без переписывания скриптов.

## 7. Риски
- Разные версии Python на машинах разработчиков.
- Отсутствие системных пакетов для некоторых зависимостей в будущем.

### Митигирующие меры
- Ввести проверку версии Python на старте.
- Добавить lock-файл зависимостей после выбора стека.

## 8. План следующих итераций
1. Добавить `requirements.txt`/`pyproject.toml`.
2. Подключить линтеры и форматтеры (`ruff`, `black`).
3. Настроить pre-commit и CI.
4. Уточнить архитектурные слои (API, domain, data).

## 9. Критерии готовности (Definition of Ready для разработки)
- [x] Есть автоматизированный скрипт подготовки окружения.
- [x] Есть документ с архитектурными решениями старта.
- [x] Есть зафиксированная структура и следующие шаги.
